GitHub Pages setup for this project
====================================

What the repo contains
- .nojekyll — disables Jekyll so GitHub serves plain static files (copied into the deploy bundle).
- tools/render_static_pages.py — reads config/site.json and English UI strings from
  locales/messages.json (strings.en only, plus site content overlays), writes a single index.html
  at the repo root. No JavaScript; no per-locale HTML files. Other languages are left to the visitor’s
  browser or OS translation tools.
- .github/workflows/pages.yml — runs the renderer, copies index.html, assets/icons/*.svg, .nojekyll,
  and optional _headers into _site, then deploys via GitHub Actions.

Build workflow (local or CI)
1. Edit config/site.json (and optionally strings.en or content / contentLocales.en in site.json).
2. Run: py tools/render_static_pages.py (Windows) or python3 tools/render_static_pages.py (Linux/macOS).
3. Commit the generated index.html together with your JSON/image changes.
4. Push; the Pages workflow re-runs the renderer so CI output always matches the committed JSON.

Logs
- The renderer appends a short line to logs/render-static-pages.log each run (UTC timestamps).

Configuration (no hand-editing of generated HTML for content)
- config/site.json
  - meta.themeColor — <meta name="theme-color"> on the page.
  - lines[] — each line:
    - id — stable identifier (used for styling fallback when accent is omitted).
    - name — large card title (human-readable service name).
    - tel — E.164 dial string for the tel: link (e.g. +442477092465).
    - display — optional smaller line under the title (national or spaced format). If empty, a
      simple UK-style national display is derived from tel when possible; otherwise tel is shown.
    - icon — relative path to an image (e.g. assets/icons/security.svg or a PNG you added).
    - subtitle — optional third line; when set, it replaces the “Tap to call” hint.
    - accent — optional visual variant: "security" | "medical-emergency" | "other". If omitted,
      security and medical-emergency ids keep their colors; everything else uses "other".
  - content — optional object of UI string keys merged on top of English (same keys as strings.en),
    e.g. pageTitle, eyebrow, heading, lead, navLabel, footerNote.
  - contentLocales.en — optional English-only overrides in the same shape as content.

- locales/messages.json
  - strings.en — English UI copy for the page (pageTitle, eyebrow, heading, lead, navLabel, tapToCall).
    Other locale entries in the file are ignored by the renderer but may be kept for reference or future use.

Published URL
- Site entry: https://<your-username>.github.io/<repo>/ (or your user site root if this repo is
  username.github.io). Use that URL in the QR code.

AWS / other static hosts
- Upload the same tree as _site: index.html, assets/icons/*, .nojekyll, and optional _headers.

One-time steps on GitHub (after you push this repo)
1. Create a new repository on GitHub (public repos get free Pages; private Pages may need a paid plan).
2. Push your local branch (main or master) to GitHub.
3. Open the repo on GitHub → Settings → Pages.
4. Under "Build and deployment", set Source to "GitHub Actions" (not "Deploy from a branch").
5. Open the Actions tab; the "Deploy GitHub Pages" workflow should run on the push. Approve Pages if prompted.

Optional: deploy from a branch instead of Actions
- Settings → Pages → Source: Deploy from branch → main (or master) → / (root).
- You must still run tools/render_static_pages.py before committing so index.html stays in sync with JSON.
- Keep .nojekyll at the repo root.

GitHub Pages setup for this project
====================================

What was added in the repo
- .nojekyll — disables Jekyll so GitHub serves plain static files (also used in the deploy bundle).
- .github/workflows/pages.yml — copies index.html, assets/*.js, config/, locales/, and .nojekyll into _site and deploys via GitHub Actions.

Configuration (no HTML edits needed for content)
- config/site.json — structure:
  - lines[] — each line: id, name, tel (E.164, e.g. +442477092465), optional display (shown under the title, e.g. national format), optional subtitle (replaces display when set).
  - meta.themeColor — sets the <meta name="theme-color"> value.
  - content — optional overrides for any UI key that matches data-i18n (e.g. lead, eyebrow, heading, pageTitle, navLabel). Non-empty values replace locales/messages.json for every language.
  - contentLocales — e.g. { "de": { "lead": "…" } } overrides for that locale only (after merging site content).
  - images — optional brand URL (relative or absolute). Empty keeps the built-in SVG.
  - content.footerNote — optional footer line (plain text).
- locales/messages.json — strings per locale, langNames for the language dropdown, rtlLocales for text direction. English includes tapToCall for the secondary line when a row has no display/subtitle.

The app loads JSON using the URL of assets/app.js to find the site root (so config/ and locales/ resolve correctly on GitHub Pages subpaths and trailing-slash edge cases).

Optional JS override: window.SECURITY_LINES — same shape as lines[] if you cannot use site.json.

One-time steps on GitHub (after you push this repo)
1. Create a new repository on GitHub (public repos get free Pages; private Pages may need a paid plan — check current GitHub pricing).
2. Push your local branch (main or master) to GitHub.
3. Open the repo on GitHub → Settings → Pages.
4. Under "Build and deployment", set Source to "GitHub Actions" (not "Deploy from a branch").
5. Open the Actions tab; the "Deploy GitHub Pages" workflow should run on the push. If it is the first time, approve Pages if GitHub prompts for it.

Your live URL
- Project site (repo named e.g. Security-Reporting-QR-Page):
  https://<your-username>.github.io/Security-Reporting-QR-Page/
- Put that URL (or /index.html) in the QR code.

Optional: deploy from a branch instead of Actions
- Settings → Pages → Source: Deploy from branch → main (or master) → / (root).
- Ensure .nojekyll exists at the repo root (it does). The workflow is then unnecessary but harmless.
